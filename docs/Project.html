teams <- teams |>
  mutate(
    WinPct = round(W/G, 4), # win percentage
    run_diff = R-RA, # run differential 
    WHIP = round((BBA + HA)/(IPouts/3), 2), # WHIP -> walks, hits per inning pitched
    AVG = round(H/AB, 3), # batting average
    OBP = round((H+BB+HBP)/(AB+BB+HBP+SF), 3), # on-base percentage 
    slugpct = round(((H-X2B-X3B-HR) + 2*X2B + 3*X3B + 4*HR) / AB, 3), # slugging percentage
    OPS = round(OBP + slugpct, 4) # OPS
  )

An observation in this data set corresponds to the data from a specific team in a specific year in baseball. There are 3614 observations in the full data set with 48 variables. I restrict the data to the 30 MLB teams over the span of time from 2021-2025 in order to see what data predicts how many wins a team gets in a season based on offensive and defensive statistics. After the changes to adjust for my time period and the necessary variables, there are 150 observations and 27 variables that have to do with offensive and defensive statistics. I have also added 7 more variables that were not included but are calculated from combinations of the given variables. These values were calculated and a small random sample was checked verifying their accuracy with manual checks on random samples.

The Society for American Baseball Research member Sean Lahman collected this data and it is updated after each season. Data is collected up to the 2025 season as recorded in the 2026 version of the database. The data was collected in order to allow people to conduct research projects and simulate games.

K-means clustering is a method that groups teams based on how similar their offensive and defensive statistics are. The model compares the teams based on the selected variables. Teams with similar values are going to be grouped together into the same cluster. The goal is to create clusters where teams are as similar as possible within the cluster and incredibly different than the teams in other clusters.

In order to do this, we have to pick how many groups we want, represented by k = 3. The algorithm randomly places 3 center points called centroids and assigns teams to the closest one based on similarities. The locations are then updated to reflect the average of the group. The process repeats many times until assignments stabilize and stop switching clusters. This helps to discover patterns that may not be obvious when looking at the data initially.

Section 4: Supervised Learning

For my models, I want to predict how many games a team would win using a k-nearest neighbors regression and a multiple linear regression model. KNN predicts the number of wins by finding statistically similar teams in the training data and averaging the total wins of the k closest teams based on the variables selected. We have to normalize these statistics because the k-nearest neighbors method depends on calculating the distance between the teams in terms of statistical distance. As you will see below, the assumption is reasonable because similar statistics tended to correlate to similar win numbers.

#split into train and test new 
set.seed(2026)
model_train <- teams_train |>
  select(W, AVG, ERA, HR, WHIP, run_diff)
model_test <- teams_test |>
  select(W, AVG, ERA, HR, WHIP, run_diff)

knn_model <- nearest_neighbor(
  mode = "regression",
  neighbors = tune()
)

# recipe 
knn_recipe <- recipe(
  W ~ ., 
  data = model_train
) |>
  step_normalize(all_numeric_predictors())

#workflow
knn_wflow <- workflow() |>
  add_model(knn_model) |>
  add_recipe(knn_recipe)

# cross validation
set.seed(2026)
cv_folds <- vfold_cv(model_train, v = 5)

# tuning 
knn_grid <- tibble(
  neighbors = seq(1, 25, by = 2)
)

knn_tune <- tune_grid(
  knn_wflow,
  resamples = cv_folds, 
  grid = knn_grid,
  metrics = metric_set(rmse)
)

# select best model and k
best_k <- select_best(knn_tune, metric = "rmse")
final_knn <- finalize_workflow(
  knn_wflow, 
  best_k
)

# fit the model

The test rmse means that my model’s prediction of team wins are off by about 4.5 wins each season. In the context of MLB, 162 games are played and wins typically vary across teams by 10s. Therefore, the model performs reasonably well in capturing overall team strength based on offensive and pitching statistics.

For this process, we tuned the number of neighbors (k) to control how many similar teams are used to make a prediction for another given team. For example, our k was 7 so the model chose the 7 most similar teams in the training data and averaged the wins to create the prediction of how many wins the team would have. In general, k controls the bias vs variance of the model. If k is small, the model is too flexible and follows the training data, which means low bias and high variance. If k is large, the model is more stable but has higher bias and lower variance since there might be some patterns missed.

Next, we are going to fit a multiple linear regression model. I used multiple linear regression because I was curious about if standard statistics can directly explain how many wins a team will get in a linear way. Multiple linear regression works by creating a single equation that represents the relationship between the predictors and the response variable (wins). It estimates how much each predictor affects the number of wins while keeping the rest of the predictors the same. This method assumes that the relationship between the predictors and response variable is approximately linear. Another assumption is that the observations are independent and that the predictors are not excessively correlated with each other. These seem to hold with my EDA and considering the variables I chose.

Conclusion

This project explored how offense and defense statistics influence a team’s success in Major League Baseball from 2021 to 2025. Stronger batting averages and lower ERA values were associated with higher winning percentages, as discovered through exploratory data analysis. Using k-means clustering, teams were grouped with similar playing styles and levels of success. Both K-nearest neighbors and multiple linear regression showed that team wins can be predicted reasonably well using team statistics. Overall, the project demonstrated that well balanced teams perform the best, and teams weaker offensively or defensively may benefit from improving those weaker areas in order to become more balanced.
